# Polynomial Server 0.2.3

## 围棋邀请与协商结算

- 接受邀请后，邀请立即从双方的“对局邀请”列表消失。
- 双方连续停一手后，KataGo 给出胜方和目数，但不立即结束棋局。
- 新增显著的全屏结算界面，明确显示黑棋或白棋获胜及胜出目数。
- 联机对局须双方分别确认结果，双方确认后才正式结束。
- 人机对局中 KataGo 一方自动确认，玩家确认后结束。
- 任一方拒绝结果时，撤回触发结算的两次停手并恢复落子。
- KataGo 裁决次数在同一棋局内累计；默认第三次裁决强制生效，不再允许拒绝。
- 管理后台新增“强制接受裁决次数”，可在 1–20 次之间调整。
- 棋局事件记录结算提议、确认、拒绝和强制生效状态。

## KataGo 配置

- 补充 KataGo 1.16.5 GTP 必需的 `resignThreshold` 和 `resignConsecTurns`，避免自动结算启动时 `SIGABRT`。
