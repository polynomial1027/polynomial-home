# Polynomial Server 0.2.1

发布日期：2026-07-21

## 围棋修复

- 入门机器人不再固定选择 KataGo 最佳点，而是在前 10 个合法候选点中随机落子。
- 初级、中级、高级候选池分别为 5、2、1，可在管理后台的难度 JSON 中通过 `candidatePool` 调整。
- KataGo 胜率连续 12 个机器人回合低于 `0.5`（50%）后，服务器自动判机器人认输；阈值和连续回合数可在后台调整。
- 禁用 KataGo 自身的提前认输，由服务器统一执行上述连续胜率规则。
- 双方连续停一手后，服务器调用 `kata-search_analyze`，以 KataGo 返回的 `scoreLead` 作为最终目数和胜负依据。
- 删除玩家标记死子、双方提交方案、接受对方方案和恢复对局的终局流程。
- 自动结算失败时棋局保留在结算状态，只允许参与者重试 KataGo 结算。
- 棋谱结果、对局记录和页面终局显示统一使用 KataGo 判定结果。

## 验证

- JavaScript 语法检查通过。
- 16 项自动化测试全部通过。
- 新增 KataGo 分析响应解析、自动点目和连续低胜率认输测试。
